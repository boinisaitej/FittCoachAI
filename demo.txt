================================================================================
 PART A - CURRENT USEFUL FEATURES (already coded in this repo)
================================================================================

[A1] ACCOUNTS & LOGIN
  - 3-role system: Gym Owner, Trainer, Client
  - First-time gym setup (only first owner can self-register)
  - Login / Logout / Change password
  - Owner creates Client accounts (with full profile fields)
  - Owner creates Trainer accounts (with specialization + type)
  - Owner can activate / deactivate any non-owner account
  - Welcome email auto-fired on account creation
        (template + wiring already in code; SMTP needs config to actually send)

[A2] PLANS (Basic / Pro)
  - Owner assigns Basic or Pro plan to a client (30-day default)
  - Owner extends subscription by N days
  - Owner applies discount %
  - Auto-extend on 7 / 30 / 90 day streak (+2 / +7 / +14 days)

[A3] CLIENT <-> TRAINER MATCHING
  - Auto-match by gender + plan rules (female->female, pro->personal)
  - Manual assign from a filtered dropdown
  - Owner reassigns trainer (old assignment kept for history)
  - Live trainer-load table (active clients per trainer)
  - Client dashboard shows their assigned trainer + quick Message

[A4] CLIENT PREFERENCES
  - Pick exercise types (chips: cardio, muscle gain, zumba, yoga, weight loss)
  - Vegetarian toggle (festivals force veg regardless)
  - AI Diet on/off, AI Workout on/off
  - Daily water goal (glasses)

[A5] DAILY PLAN (todos)
  - Trainer builds today's plan (food / exercise / water / sleep rows)
  - Client checks off todos (Completed / Partial / Skipped)
  - "Why is this in my plan?" AI explanation popover per row
  - Trainer grants a cheat day

[A6] CALENDAR
  - Month grid with green/yellow/red dots
  - Click date -> day modal with status buttons
  - 7-day streak strip on dashboard

[A7] DASHBOARDS & CHARTS
  - Client dashboard (BMI, points, charts, 7-day streak, reports)
  - Trainer dashboard (KPIs, alerts, auto-progression, broadcast, client grid)
  - Trainer -> Client deep page (Overview / History / Recommendation /
    Notes / Send report / Send notification)
  - 5-second live refresh on critical views

[A8] BMI MONITORING
  - Log BMI (auto-classification + delta vs previous)
  - BMI history line chart + table on client dashboard

[A9] HEALTH ISSUES (AI remedies)
  - Submit problem + severity -> AI returns foods/exercises/tips
  - Severe reports raise trainer alerts

[A10] CHAT
  - Three-pane chat (role-aware sidebar)
  - Trainer <-> Client + Owner <-> any + AI tab
  - AI Chat with sessions sidebar + history (Pro = unlimited,
    Basic = 10/day, resets at midnight)
  - Right-click bubble: Copy / Delete
  - Trainer broadcast to all clients (chat + bell)

[A11] AI-GENERATED PLANS
  - AI Diet (single day) - 4 meal cards w/ recipe pencil
  - AI Diet (full week)
  - Regenerate single meal with feedback
  - AI Workout (single day + full week)
  - Apply-to-today one-click
  - Injury-aware filtering
  - AI motivational ping banner
  - AI Recipe on demand (Pro)
  - AI Food Scanner (Pro)
  - AI auto-progression proposals every Monday 06:00

[A12] NUTRITION KNOWLEDGE BASE
  - FAISS-backed RAG search + history sidebar
  - Source snippets shown with score
  - "Vector live" vs "Fallback" indicator

[A13] AI GROCERY LIST
  - Generate from date range -> grouped by category + INR totals
  - History sidebar (Rename / Duplicate / Delete)
  - Checklist mode with progress bar

[A14] JUNK FOOD / WATER / SLEEP TRACKERS
  - Log junk food (weekly trend + trainer alert on threshold)
  - Water +1/-1 quick buttons + week bar chart
  - Sleep hours + notes (sleep_low alert if < 6h)
  - Trainer alerts inbox (sleep_low / severe_health / injury / junk_excess)

[A15] INJURIES
  - Client reports injury (tag + severity + notes)
  - Future AI workouts auto-exclude conflicting exercises
  - Trainer resolves -> re-include exercises

[A16] PROGRESS PHOTOS
  - Upload Before / Progress / After (with notes)
  - Before vs After side-by-side compare card

[A17] INVOICES & REPORTS (PDFs)
  - Owner issues invoice -> client gets bell + PDF in profile
  - Weekly adherence report (Pro) emailed/listed in client dashboard
  - Trainer weekly summary aggregator (saved AI paragraph)

[A18] NOTIFICATIONS, ANNOUNCEMENTS, EMAIL
  - In-app bell + /notifications page (bulk delete, mark read, clear)
  - Trainer immediate / scheduled / recurring notifications
  - Owner announcement (audience + scheduled) - bell + email
  - Welcome email + plan-assigned email (templates exist)
  - Web push (VAPID scaffold) - reminds even with tab closed

[A19] GAMIFICATION
  - Points per todo (10 / 4)
  - Streak milestone rewards (7/30/90-day -> subscription extension)

[A20] RATINGS
  - Weekly mutual rating (client rates trainer + trainer rates client)

[A21] FESTIVALS / VEG ENFORCEMENT
  - Festival-aware diet banner + forced veg on those dates

[A22] SLOGAN BANNER
  - Animated slogan strip on every page (22 defaults + AI-generated)

[A23] PROFILE PAGE
  - Single template, role-tailored Key/Value table
  - Owner edits gym name + address (propagates to invoices + emails)
  - Client "My invoices" tab

[A24] OWNER CONSOLE
  - KPI row + numbered step-by-step cards (create, assign, reassign,
    trainer load, all users, invoices, announcements, slogans)

[A25] BACKGROUND JOBS
  - Every-minute scheduler for announcements + trainer notifications
  - Monday 06:00 auto-progression proposals
  - KB index warm-up on startup
  - Sleep / junk / stale-todo / weekly-report monitors

[A26] DEMO TOOLS
  - Seed-demo command + smoke-test scripts

================================================================================
 PART D - CONFIG CHANGES TO TURN EMAIL ON RIGHT NOW (zero new code)
================================================================================

Append to .env:

    MAIL_ENABLED=1
    SMTP_HOST=smtp.sendgrid.net        (or smtp.gmail.com for a quick test)
    SMTP_PORT=587
    SMTP_USER=apikey                   (literal word "apikey" for SendGrid)
    SMTP_PASS=<sendgrid-api-key>
    MAIL_FROM=FitCoachAI <noreply@yourdomain.com>
    APP_NAME=FitCoachAI

After saving + restarting Flask:
  - Owner creates Trainer  -> trainer receives "your account is ready" email
  - Owner creates Client   -> client receives "your account is ready" email
  - Trainer assigns plan   -> client receives "new plan for <date>" email
  - Owner announces        -> recipients receive "announcement" email
